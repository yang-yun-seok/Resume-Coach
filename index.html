Start Here

처음 들어오셨다면, 이 설명서부터 보고 시작하시면 됩니다.

설정에서 API를 연결하고, 지원 직무를 정한 뒤, PDF 또는 텍스트를 넣으면 오른쪽 패널에 분석 결과가 생성됩니다.

1. 설정 연결

Gemini API Key와 모델을 먼저 입력합니다.

2. 타깃 지정

기업과 직무를 넣으면 평가 관점이 더 선명해집니다.

3. 자료 투입

PDF를 올리거나 초안 텍스트를 붙여 넣습니다.

4. 결과 해석

요약, STAR 구조, 보완 포인트를 보고 문서를 다듬습니다.

가장 좋은 사용 순서

  • 상단 설정 버튼에서 API Key와 모델을 입력합니다.
  • 연결 테스트로 실제 응답 여부를 확인합니다.
  • 지원 회사와 직무를 입력한 뒤 PDF 또는 텍스트 모드를 선택합니다.
  • AI 분석 시작하기를 누르면 오른쪽에 리포트가 생성됩니다.
  • 리포트를 기준으로 자소서와 포트폴리오 문장을 다시 다듬습니다.

결과를 읽는 법

  • 직무 경험 요약은 지금 문서가 어떤 인재로 읽히는지 보여줍니다.
  • STAR 구조화는 면접 답변으로 바로 전환 가능한 핵심 경험을 정리합니다.
  • 보완 포인트는 서류 통과를 위해 우선 보강할 빈칸을 알려줍니다.

입력 팁

  • 프로젝트 목적, 팀 규모, 역할, 성과 수치가 들어가면 품질이 높아집니다.
  • 메모 형태라도 괜찮지만 경험 사이 구분을 해두면 정확도가 좋아집니다.
  • PDF는 텍스트 선택이 되는 문서일수록 안정적으로 분석됩니다.

이럴 때 특히 좋습니다

  • 포트폴리오 설명이 직무 관점으로 읽히는지 점검하고 싶을 때
  • 자소서 초안이 감정형 표현에 치우쳤는지 확인하고 싶을 때
  • 면접 대비용으로 경험을 STAR 구조로 정리하고 싶을 때

주의사항

API Key는 브라우저에서 직접 Gemini API로 전송됩니다. GitHub Pages처럼 정적 배포에는 바로 쓸 수 있지만, 공개 서비스 운영에는 백엔드 프록시를 두는 편이 더 안전합니다.
